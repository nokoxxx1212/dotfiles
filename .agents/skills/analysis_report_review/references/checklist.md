# 分析報告書（レポート） スライド構成チェックリスト

本報告書は、設計書から始まり最終報告書へと進化する「スライド資料」です。
各スライドには必ず **【タイトル（章名）】【キーメッセージ（1〜2行）】【本文（箇条書き）】** を含め、5W1Hを意識してMECEかつシンプルに整理してください。情報過多の場合は、2階層（例：2-1, 2-2）にスライドを分割してください。

## 📌 フェーズ1：分析着手前に埋めるスライド（設計部分）

### 1. 分析概要
**1-1. 与件整理（背景と目的）**
- **キーメッセージ**: なぜこの分析を行うのか、何を明らかにするのかを1行で表現。
- **本文の必須観点**:
  - 現状の課題と理想のギャップ。
  - 定量的な効果見込み（この分析をすることのビジネス上の意義）。
  - **分析目的の解像度（NGワード禁止）**：「XXの分析」といった手段の目的化（曖昧な表現）を厳しく禁止し、「XXの明確化」「XXの検証」といった白黒つく具体的な表現へ修正させる。
  - **指標の厳密性**: 評価に使用する指標（KPI等）の、分母・分子・集計期間・除外条件の明確な定義。

**1-2. イシューツリーと検証仮説**
- **キーメッセージ**: 解決すべき最大の「問い」と「初期仮説」を1行で表現。
- **本文の必須観点**:
  - **メインイシュー**: 解決すべき最大の「問い」。
  - **サブイシュー**: メインイシューを分解したサブイシュー群。必ず**「〜か？」という問いの形式**にすること。
  - **初期仮説**: 各問いに対する結論の仮置き。

**1-3. 実施概要とデータ要件**
- **キーメッセージ**: どのような対象に、どうアプローチするのかを1行で表現。
- **本文の必須観点**:
  - **アプローチ手法とその選択理由**: 分析・比較設計（A/Bテスト等）の手法と、なぜその手法を選んだかの理由。
  - **データと抽出条件**: 対象者条件、除外条件、利用するデータソース。
  - **統計的妥当性とバイアスチェック**: 十分な**サンプルサイズ**が確保できる見込みか。データ抽出時に生じる**バイアス（偏り）リスク**への対処方針。

---

## 📌 フェーズ2：分析結果が出た後に埋めるスライド（報告部分）

### 2. 分析結果
**2-1. 結果のまとめ（全体像）**
- **キーメッセージ**: 分析を通じて得られた最大の発見（ファクトに基づく結論）を1行で表現。
- **本文の必須観点**:
  - テーマごとの総括。
  - **「事実（空）→解釈・洞察（雨）→行動・提案（傘）」**のフレームワークで全体をまとめる。

**2-2-X. 分析結果サマリ（複数スライド）**
※サブイシュー（問い）ごとに1枚のスライドを作成する。
- **タイトル**: 必ず**「問いの形式」**にする（例：「ロイヤル顧客の離脱理由は価格か？」）。
- **キーメッセージ**: その問いに対する明確な結論。
- **本文の必須観点**:
  - **【必須】空・雨・傘の徹底**: スライド単体で「客観的なファクト・グラフ（空）」「分析者の解釈・洞察（雨）」「ネクストアクションへの示唆（傘）」が揃っているかを厳しくチェックする。事実の羅列だけで終わっている場合は修正を要求する。
  - **ロジカル接続性と因果のチェック**:
    - 前後のスライドが論理的に接続詞で繋がっているか（ストーリーラインの担保）。
    - **「相関関係」を「因果関係」と取り違えた解釈をしていないか**。
    - 確証バイアスなど、結果に対する解釈の偏りがないか。

### 3. 今後に向けた提案
**3-1. ネクストアクション**
- **キーメッセージ**: 分析結果を踏まえ、事業部が次に取るべき具体的なアクションを1行で表現。
- **本文の必須観点**:
  - **「現状→考察→対応策」**の枠組みで提言をまとめる。
  - **現実感への落とし込み**: 理論解（データ上の正解）をそのまま提案するのではなく、現場の運用や新たな制約条件を発見・考慮した現実的なアクションに落とし込まれているかをチェックする。
