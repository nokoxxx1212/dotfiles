# Step 3: データ収集・前処理（Data） 実行チェックリスト

分析プロジェクトにおける「Step 3: データ収集と前処理」を自律的に実行する際、以下のベストプラクティスとチェックポイントを満たすようにコードを記述・実行してください。

## 1. 抽出条件と安全性の確認
- **スキャン量チェック（必須）**: SQLを実行する前に、必ずDry-RunやEXPLAINでコスト・データ量を見積もり、閾値（1GB等）を超える場合はユーザーに承認を求めているか。
- **除外条件の適用**: テストアカウント、社内ユーザー、システム異常値など、分析ノイズとなるデータを除外する処理が入っているか。
- **Fan-outの防止**: テーブル結合（JOIN）時に、意図せずレコード数が倍増（Fan-out）していないか、一意性（ユニークキー）を確認しているか。

## 2. 基礎プロファイリング（データ品質チェック）
- **欠損値（NULL）**: 各カラムの欠損割合を算出し、補完や除外の判断を行っているか。
- **異常値・外れ値**: 年齢がマイナス、購入金額が異常に大きいなど、ビジネスロジック的にあり得ない値を検知しているか。
- **分布の確認**: 主要な変数の分布（ヒストグラム）やカーディナリティ（カテゴリの種類数）を確認し、分析に十分なサンプルサイズがあるかを評価しているか。

## 3. 分析用データセットへの前処理
- **フォーマット変換**: 日付型の統一、カテゴリ変数のエンコーディングなど、次のステップ（Step 4）で扱いやすい形式に変換しているか。
- **特徴量作成（必要に応じて）**: 既存カラムから、Issue解決に役立つ新しい変数（例：経過日数、フラグ）を作成しているか。

## 4. サマリーの言語化
- 単にコードを実行するだけでなく、人間が読んで理解できる「データ品質サマリー」として、データの使える点・使えない点（限界）を言語化して出力すること。
