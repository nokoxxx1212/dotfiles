# 分析報告書（レポート） スライド構成チェックリスト

本報告書は、設計書から始まり最終報告書へと進化する「スライド資料」です。
各スライドには必ず **【タイトル（章名）】【キーメッセージ（1〜2行）】【本文（箇条書き）】** を含め、5W1Hを意識してMECEかつシンプルに整理してください。情報過多の場合は、2階層（例：2-1, 2-2）にスライドを分割してください。

## 📌 フェーズ1：分析着手前に埋めるスライド（設計部分）

### 1. 分析概要
**1-1. 与件整理（背景と目的）**
- **キーメッセージ**: なぜこの分析を行うのか、何を明らかにするのかを1行で表現。
- **本文の必須観点**:
  - 現状の課題と理想のギャップ。
  - 定量的な効果見込み（この分析をすることのビジネス上の意義）。

**1-2. イシューツリーと検証仮説**
- **キーメッセージ**: 解決すべき最大の「問い」と「初期仮説」を1行で表現。
- **本文の必須観点**:
  - **メインイシュー**: 解決すべき最大の「問い」。
  - **サブイシュー**: メインイシューを分解したサブイシュー群。必ず**「〜か？」という問いの形式**にすること。
  - **初期仮説**: 各問いに対する結論の仮置き。

**1-3. 実施概要とデータ要件**
- **キーメッセージ**: どのような対象に、どうアプローチするのかを1行で表現。
- **本文の必須観点**:
  - アプローチ手法、対象者条件、除外条件。
  - 利用するデータソース（テーブル名など）。

---

## 📌 フェーズ2：分析結果が出た後に埋めるスライド（報告部分）

### 2. 分析結果
**2-1. 結果のまとめ（全体像）**
- **キーメッセージ**: 分析を通じて得られた最大の発見（ファクトに基づく結論）を1行で表現。
- **本文の必須観点**:
  - テーマごとの総括。
  - **「事実（空）→解釈・洞察（雨）→行動・提案（傘）」**のフレームワークで全体をまとめる。

**2-2-X. 分析結果サマリ（複数スライド）**
※サブイシュー（問い）ごとに1枚のスライドを作成する。
- **タイトル**: 必ず**「問いの形式」**にする（例：「ロイヤル顧客の離脱理由は価格か？」）。
- **キーメッセージ**: その問いに対する明確な結論。
- **本文の必須観点**:
  - グラフや集計結果から読み取れる客観的なファクト（事実）。
  - ※前後のスライドを接続詞で繋ぎ、全体で1つのストーリーにする。

### 3. 今後に向けた提案
**3-1. ネクストアクション**
- **キーメッセージ**: 分析結果を踏まえ、事業部が次に取るべき具体的なアクションを1行で表現。
- **本文の必須観点**:
  - **「現状→考察→対応策」**の枠組みで提言をまとめる。
