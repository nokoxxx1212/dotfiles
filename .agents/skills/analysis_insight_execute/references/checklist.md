# Step 4: 分析（Analysis） 実行チェックリスト

分析プロジェクトにおける「Step 4: 分析（深い洞察の抽出）」を自律的に実行する際、以下のベストプラクティスとチェックポイントを満たすようにアプローチし、コードを実行してください。

## 1. プロファイリング結果に基づく前処理
- **申し送り事項の確認**: Step 3で作成されたプロファイリングレポート（欠損値、ノイズ、ビジネスルールの制約）を熟読し、分析に与えるリスクを把握しているか。
- **必要な前処理の実行**: 上記のリスクおよび自身の分析設計に基づき、テーブルの結合（JOIN）、フォーマット変換、新しい特徴量の作成などの高度な前処理をNotebook上で確実に実行しているか。

## 2. 適切な「比較視点」の設計
意味合い（Insight）は常に「比較」から生まれます。Issueに答えるために、以下のどれ（または複数）の比較視点を用いるか明確に定義して実行しているか。
- **A/B比較**: 介入群と統制群、新機能利用ユーザーと未利用ユーザーの比較。
- **時系列比較**: 施策前後、前年同期比、トレンドの変曲点の比較。
- **セグメント比較**: 新規vs既存、ロイヤル層vs離脱層など、属性による切り分け。

## 3. 統計的妥当性の担保
- **サンプルサイズの確認**: 比較を行う際、それぞれの群に十分なデータ量（n数）があるかを確認しているか。
- **交絡因子の排除（擬似相関の警戒）**: 「Aが高いからBが高い」という相関関係を、安易に因果関係（AをやればBが上がる）と取り違えていないか。他の要因が影響していないか留意したコード・解釈になっているか。
- **有意差とビジネスインパクトの区別**: 統計的に有意であっても、ビジネス上のインパクト（効果量）が微小であれば「意味がない」と正しく解釈できているか。

## 4. 「深いSo What（洞察）」の抽出
単なる「観察事実」をアウトプットとするのは禁止です。一段深く掘り下げてください。
- ❌ **観察レベルのSo What**: 「20代女性のCVRが先月より15%低下しています。」
- ⭕ **洞察レベルのSo What**: 「20代女性のCVR低下は、直近のUI変更により決済画面での離脱が特定のOSで急増したことが要因です。したがって、当該OSのUI改善を最優先で行うべきです。」

## 5. ビジネス向け可視化（Storytelling with Data）の徹底
分析結果をグラフ化する際は、以下のアンチパターンを厳格に排除し、意思決定者が一目で理解できる可視化を行うこと。
- **アンチパターンの禁止**: 要素が多すぎる円グラフ、不要な3Dグラフ、二重軸（Dual Axis）の乱用を禁止する。
- **データインク比の最大化**: メッセージに不要なグリッド線や背景色は削除する。
- **タイトルとメッセージの連動**: グラフのタイトルには「単なる変数名」ではなく、「そのグラフから何が言いたいか（結論）」を記載する。

## 6. ビジネスアクションへの接続
- 分析結果が、「だから私たちは明日から〇〇すべきである」という具体的な提案・ネクストアクションに直接繋がる形で言語化されているか（※制約条件が不明な場合は、前提条件を置いた上で提言を行うこと）。
- **理論解から現実感への落とし込み**: データから導かれた「理論上の正解（理論解）」をそのまま無邪気に提案するのではなく、それを現場の運用に当てはめた場合に発生しうる「新たな制約条件」や「オペレーションの壁」をリスクとして考慮し、現実感のあるアクションに落とし込めているか。
